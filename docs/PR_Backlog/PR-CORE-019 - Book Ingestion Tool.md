PR-CORE-019 – Book Ingestion Tool

Status: Specification
Priority: HIGH
Effort: MEDIUM
Phase: Post‑Unification Core Refinement
Date: 2026‑03‑29

Context & Motivation

A primary long‑term goal of StableNew is to transform entire books into cinematic experiences. To achieve this, we must ingest a book’s text and break it down into scenes and shots (see PR‑CORE‑003) and subsequently generate visuals. Currently, scene planning must be triggered manually by the user or via a limited CLI. A Book Ingestion Tool will automate the first part of this process: loading a book (e.g. a novel like The Way of Kings) and producing an initial StoryPlan composed of chapters, scenes, and shots. This scaffolding will integrate with the scene planning module and prompt pack builder.

Goals & Non‑Goals
Goal: Provide a command‑line and/or GUI tool that accepts a book file (plain text, ePub, or PDF) and outputs a structured representation (chapters -> scenes -> shots) stored as a StoryPlan in JSON/YAML format.
Goal: Use natural language processing (NLP) techniques or LLMs to detect scene boundaries, major events, and break them into smaller shot descriptions. Provide reasonable defaults but allow user editing.
Goal: Integrate the ingestion tool with the existing story_plan_store.py so that the resulting plan can be loaded into the scene planner UI.
Goal: Support incremental ingestion: re‑run on individual chapters or sections without reprocessing the entire book.
Non‑Goal: Performing actual video generation; this PR focuses solely on text ingestion and plan creation.
Guardrails
Respect copyright: do not distribute book contents. The ingestion tool should process user‑provided files locally and store only the structured metadata (scene titles and descriptions), not the raw text.
Provide clear logging or progress indicators during ingestion. Users should know which chapter is being processed and how far along the tool is.
Ensure the tool gracefully handles poorly formatted books or unexpected file formats by failing with informative error messages.
Allowed Files
Files to Create
src/tools/book_ingester.py – a command‑line interface (CLI) script that accepts a file path to a book and outputs a StoryPlan JSON file. It will orchestrate reading the file, splitting chapters, calling a scene segmentation engine, and generating shot plans.
src/nlp/scene_segmenter.py – a module that uses heuristics or calls an LLM to break text into scenes. It exposes a function segment_chapter(text: str) -> List[str] returning scene descriptions.
src/nlp/shot_generator.py – optional module that takes a scene description and splits it into a list of shot prompts. This can be a simple heuristic (e.g. one shot per sentence) or call an LLM.
src/video/story_plan_loader.py – functions to convert lists of scenes and shots into the existing StoryPlan dataclasses and save them to disk using the story_plan_models.py structures.
tests/tools/test_book_ingester.py – unit tests for ingesting small dummy books and verifying that the output StoryPlan has the correct number of scenes and shots.
Files to Modify
src/video/story_plan_store.py – add methods to import plans from JSON files generated by the ingestion tool. Ensure that the StoryPlan dataclasses include fields for chapter_number, scene_index, etc., if not already present.
src/gui/views/story_plan_tab_frame.py – optionally add a button to import a book and view the automatically generated plan.
Forbidden Files
Do not modify core pipeline code for video generation. The ingestion tool is purely a pre‑production aid.
Implementation Plan
Book Parsing: In book_ingester.py, detect the file type (plain text, ePub, PDF). Use existing libraries (e.g. ebooklib for ePub, PyPDF2 for PDF) to extract chapter text. For large novels, implement streaming parsing to avoid loading the entire file into memory.
Scene Segmentation: Implement scene_segmenter.py with two modes: a rule‑based heuristic (e.g. splitting on chapter breaks, dialogue transitions, or scene separators) and an LLM‑powered segmentation. For the LLM mode, call a local or external model (mocked in tests) with a prompt instructing it to summarise each scene. Return a list of scene descriptions.
Shot Generation: Optionally, implement shot_generator.py to break each scene description into multiple shot prompts. Use heuristics (e.g. one shot per sentence) or LLM summarisation. Provide configuration options (e.g. number of shots per scene). This step can be deferred or simplified if not needed immediately.
Plan Construction: Use story_plan_loader.py to convert the extracted scenes and shots into a StoryPlan object. Fill fields like title, scene_index, shots, and anchors. Save the plan to JSON using the existing to_dict function in StoryPlan.
CLI: Provide a CLI interface (e.g. via argparse) that accepts parameters: input file path, output directory, segmentation mode, number of shots per scene, and whether to run shot generation. Print progress messages (e.g. “Processing Chapter 5/50”).
Integration with UI: Modify story_plan_store.py and the scene planner tab so that imported plans show up in the UI. Add a file selection dialogue in the scene planner to load a plan created by the ingestion tool.
Testing: Write unit tests in test_book_ingester.py that use short sample texts. Verify that the number of scenes matches expectations and that the plan can be loaded into the StoryPlan dataclass without errors.
Documentation: In PR‑CORE‑018’s docs, write a tutorial on using the book ingestion tool to convert a chapter into a story plan.
Testing Plan
Unit Tests: Test parsing of different file formats (plain text, ePub, PDF) and ensure that the scene segmentation returns expected lists. Test that the CLI outputs a valid StoryPlan JSON and that story_plan_store.py can import it.
Integration Tests: Create a small dummy book with two chapters and run the ingestion tool end‑to‑end. Verify that the resulting plan has the correct structure (chapters, scenes, shots) and can be visualised in the UI.
Performance Tests: For larger files, measure ingestion time and memory usage. Ensure that streaming parsing prevents memory explosions.
Verification Criteria
The ingestion tool processes books in at least three formats (txt, ePub, PDF) and outputs a StoryPlan JSON without errors.
Scene segmentation creates a reasonable number of scenes per chapter, and shot generation (if enabled) yields multiple shots per scene.
Plans created by the tool are compatible with StoryPlan dataclasses and can be loaded into the scene planner UI.
Unit and integration tests pass, demonstrating robustness and correctness.
Documentation includes a tutorial covering the ingestion tool.
Risk & Mitigation
Risk: NLP/LLM segmentation may produce inconsistent results. Mitigate by providing a rule‑based fallback and exposing parameters for scene length or shot count.
Risk: Handling large book files may be resource intensive. Use streaming and incremental parsing, and allow users to ingest chapters individually.
Risk: Copyright concerns around book ingestion. Ensure the tool stores only metadata and avoid reusing text excerpts in outputs beyond fair use.
Dependencies
Depends on PR‑CORE‑003 (scene planning) to define StoryPlan structures and on PR‑CORE‑018 to document usage.
Approval & Execution
Approvers: NLP team, project maintainers.
Execution: Merge into a feature/book-ingestion branch after review. Ensure that dependencies (eBook parsing libraries) are added to requirements. After merging, coordinate with docs team to update tutorials.
